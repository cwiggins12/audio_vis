Piecemeal implementation order

1. Extract the active shader into a ShaderPreset struct with hardcoded spec and the current test shader. Nothing changes behaviorally, you're just wrapping what exists. Verify the test still runs identically.
2. Add a std::vector<ShaderPreset> and an activeIdx int. Put two copies of the test preset in the list and wire the right arrow key to cycle activeIdx and call your existing spec swap. Verify the key press works and the bridge reconfigures without crashing.
3. Implement the introspection-based uniform map on Shader or ShaderPreset. Replace all the hardcoded glGetUniformLocation calls in main with lookups into the map. Verify everything still renders.
4. Write the injected header and the Shader constructor change that prepends it before compiling. Verify the test shader still compiles with the header injected (it will conflict until you remove the duplicate buffer declarations from fragmentSrc).
5. Write the flat config file parser for AudioSpec. Point it at a hand-written file next to your test shader. Verify the parsed struct matches what you wrote in the file.
6. Write the directory scanner that produces a vector<ShaderPreset> from a shaders/ folder. Move the test shader there, verify it loads and runs exactly as before.
7. Write a second real shader using only the injected helper functions. Drop it in shaders/. Verify cycling between the two works end to end including the spec swap and SSBO resize.

Hot-reload and error overlays are both good follow-ons after step 7.
