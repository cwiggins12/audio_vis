Done This Week:
av_bridge. Massive changes to audio.h, main.cpp, analysis.h, and a bit to capture.h to account for the changes. All configurable variables in fft, rms, and peak can now be affected by a struct and a call with that struct to audio and avBridge.

avBridge contains control over the gpu arrays to send and how they are formatted, all of which can be dynamically adjusted by the AudioSpect struct that is meant to be paired with each fragment shader. 

AudioSpec allows for dynamic changes to these variables according to the fragment shader creators needs: an arbitrary size that will give a non noisy response curve, smoothing over time with variable attack and release for fft, rms and peak, the full fft output, the fft's audible bin output only, peak holds, fft bin holds, mono for peak and rms, values in db or magnitudes on all analysis output, single sided for fft, hann windowing for fft, perceptual slope for fft at any chosen float, along with handling with and height dependencies for the fft output for the user on window resizes.
